# /add-page 自動実行ルール

以下のいずれかが発生した場合、実装完了後に **必ず `/add-page` を自動実行**すること。

## トリガー条件

| 変更内容 | 例 |
|---------|-----|
| 新しいページ・機能を追加した | コントローラー・ビュー・サービスの新規作成 |
| DB テーブル・カラムを追加・変更した | エンティティクラスの追加・変更、DBContext の変更 |
| 技術スタック・依存ライブラリを変更した | NuGet パッケージの追加・変更 |
| プロジェクト構成を変更した | フォルダ追加、名前空間変更 |

## 実行タイミング

以下の順番で実行すること：

1. agent-browser デバッグ（動作確認）
2. **Evaluator レビュー**（`superpowers:code-reviewer` → `evaluator.md` 参照）
3. `/add-page` 実行

> **注意**: `/add-page` はホーム画面・ナビバー・ドキュメント・README・コミットまでを一括チェックするスキル。
> Evaluator レビューで差し戻しがない状態になってから実行すること。
