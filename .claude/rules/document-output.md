# ドキュメント出力ルール

## doc/ フォルダの構成

| ファイル | 形式 | 内容 |
|---------|------|------|
| 基本設計書 | .md / .docx | システム概要・機能一覧・画面一覧・非機能要件 |
| 詳細設計書 | .md / .docx | コントローラー・サービス・リポジトリ・エンティティ設計 |
| DB設計書 | .md / .xlsx | テーブル定義・Enum定義・初期データ |
| テスト設計書 | .md / .docx | テスト方針・テスト観点・実行方法 |
| テストケース一覧 | .xlsx | テストケース管理表 |
| API仕様書 | .md / .docx | 内部 Ajax API エンドポイント仕様 |
| 画面設計書 | .md / .docx | 全画面の共通仕様・各画面要素・操作フロー |
| デプロイ手順書 | .md / .docx | IIS デプロイ手順・ロールバック |
| CommonLibrary利用ガイド | .md / .docx | CommonLibrary の利用方法・コード例 |

## 更新ルール

- **Markdown（.md）が正**。Office ファイルは派生物。
- 機能追加・変更時は **Markdown を先に更新** → `/export-docs` で Office 再生成。

## トリガー（変更内容 → `/export-docs` で更新するファイル）

| 変更内容 | 更新ファイル |
|---------|------------|
| 機能追加・画面追加 | 基本設計書・詳細設計書・画面設計書 |
| テーブル・カラム追加・変更 | DB設計書 |
| テスト追加・変更 | テスト設計書・テストケース一覧 |
| API エンドポイント追加・変更 | API仕様書 |
| デプロイ手順の変更 | デプロイ手順書 |
| CommonLibrary の変更 | CommonLibrary利用ガイド |

## 形式選択（新規ドキュメント追加時）

- 説明文・手順・コード例が主体 → Word
- テーブル定義・テストケースなどデータが主体 → Excel
- 両方 → Word + Excel の 2 ファイル
