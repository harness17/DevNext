# Evaluator（コードレビュー）ルール

Playwright デバッグ完了後、`superpowers:code-reviewer` エージェントでコードを評価すること。

## 実行順序

Playwright デバッグ → **Evaluator レビュー** → `/add-page`

## 評価軸と重み

| 軸 | 重み | 確認内容 |
|----|------|---------|
| セキュリティ | ⭐⭐⭐ | 認可チェック・XSS・SQL インジェクション・CSRF |
| 保守性 | ⭐⭐⭐ | コメント量・命名・単一責任・`coding-policy.md` 準拠 |
| テスト品質 | ⭐⭐ | 境界値・異常系・モックの適切な使用 |
| 機能完成度 | ⭐⭐ | スプリントコントラクトの完成条件をすべて満たしているか |

## 指示テンプレート

```
superpowers:code-reviewer を使って以下を評価してください：
- 対象：〇〇機能の実装
- 重点：セキュリティ（認可）・保守性（コメント・命名）
- スプリントコントラクトの完成条件：[条件リスト]
```

## 差し戻し基準

- 認可チェックの漏れがある
- `coding-policy.md` のコメントルールに著しく違反している
- スプリントコントラクトの完成条件を満たしていない
