# スプリントコントラクト（実装前合意）ルール

実装を開始する前に、必ず「完成条件」を宣言すること。

## 適用タイミング

- 新機能・新ページを実装するとき
- バグ修正で影響範囲が広いとき
- リファクタリングで複数ファイルを変更するとき

## フォーマット

```
【スプリントコントラクト】
実装内容：〇〇機能を追加する
完成条件：
- 条件1（正常系）
- 条件2（権限・認可）
- 条件3（異常系・エラー処理）
→ 上記を確認後に実装開始
```

## 注意

- 完成条件は実装前に確定する（実装しながら変えない）
- 曖昧な条件は「〇〇したとき〇〇になる」と具体化する
- Playwright デバッグ時にこの条件リストをそのまま検証チェックリストとして使う
