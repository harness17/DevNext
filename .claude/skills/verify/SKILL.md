以下の手順でビルド・テスト・ブラウザ確認を実行し、結果を報告してください。

## 手順

### Step 1: ビルド確認

```bash
cd H:/ClaudeCode/DevNext && dotnet build DevNext.sln
```

ビルドエラーがあれば内容を報告し、修正を提案する。エラーがあれば以降の手順は実行しない。

### Step 2: xUnit テスト実行

```bash
cd H:/ClaudeCode/DevNext/Tests && dotnet test
```

テストの合否・件数・失敗内容を報告する。テスト失敗があれば原因を報告し、以降の手順は実行しない。

### Step 3: アプリ起動確認

agent-browser でブラウザ確認を行う前に、アプリが起動しているか確認する。

```bash
# ポート 5232 が LISTEN しているか確認
netstat -an | grep 5232 | grep LISTEN
```

起動していない場合はユーザーに通知する：

```
アプリが起動していません。別ターミナルで以下を実行してください：
  cd H:/ClaudeCode/DevNext/DevNext && dotnet run
起動後に /verify を再実行してください。
```

### Step 4: ブラウザ確認（agent-browser）

`agent-browser` で `http://localhost:5232` にアクセスして実装した機能を動作確認する。

**必須チェック項目（すべて確認すること）:**

| # | 確認項目 | 操作 |
|---|---------|------|
| 1 | ホーム画面の表示 | `http://localhost:5232` にアクセス |
| 2 | ログイン動作 | `admin1@sample.jp` / `Admin1!` でログイン |
| 3 | 実装した機能の正常系 | 追加・変更した機能を一通り操作 |
| 4 | 認可チェック | ログアウト状態で保護ページにアクセス → リダイレクト確認 |
| 5 | エラー表示 | バリデーションエラーや異常系を操作 |

**実装内容に応じて追加確認:**
- DB 操作（作成・更新・削除）が実際に反映されているか
- Admin 限定ページが Member ロールでアクセス拒否されるか
- 検索・ページング・ソートが正しく動作するか

## 報告形式

```
【verify 結果】
Step 1 ビルド: ✅ 成功 / ❌ 失敗（エラー内容）
Step 2 テスト: ✅ X件合格 / ❌ X件失敗（失敗内容と原因）
Step 3 アプリ: ✅ 起動中 (5232) / ⚠️ 未起動
Step 4 ブラウザ:
  - ホーム画面: ✅
  - ログイン: ✅
  - [機能名] 正常系: ✅ / ❌（問題内容）
  - 認可チェック: ✅
  - エラー表示: ✅
```

全ステップ ✅ の場合のみ「verify 完了」と報告する。
