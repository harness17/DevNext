## 2. 前提条件

### サーバー要件

| 要件 | 確認方法 |
|------|---------|
| .NET 10.0 ランタイムがインストール済み | `dotnet --version` |
| IIS がインストール・起動済み | IIS マネージャー確認 |
| IIS の `AspNetCoreModuleV2` がインストール済み | .NET Hosting Bundle インストール後に確認 |
| SQL Server が起動済み | SQL Server Management Studio で接続確認 |
| `DevNext` AppPool が作成済み | IIS マネージャー > アプリケーション プール |

### AppPool 設定

| 設定項目 | 値 |
|---------|-----|
| .NET CLR バージョン | マネージ コードなし |
| マネージ パイプライン モード | 統合 |
| ID | ApplicationPoolIdentity または専用サービスアカウント |

## 3. IIS デプロイ手順

### Step 1: Release ビルド

```bash
# DevNext/ ディレクトリで実行
cd DevNext
dotnet publish -c Release -o C:/deploy/DevNext
```

ビルド成果物は `C:/deploy/DevNext` に出力される。

**ビルド成果物の確認事項**:
- `DevNext.exe` が存在すること
- `web.config` が生成されていること
- `wwwroot/` フォルダが含まれていること

---

### Step 2: AppPool の停止

IIS で稼働中のアプリを安全に停止する。

```powershell
# PowerShell（管理者権限で実行）
Import-Module WebAdministration
Stop-WebAppPool -Name 'DevNext'
Write-Host "AppPool 停止完了"
```

AppPool の状態確認：

```powershell
Get-WebAppPoolState -Name 'DevNext'
# 出力: Stopped
```

---

### Step 3: ファイルの配置

既存ファイルを新しいビルド成果物で上書きする。

```powershell
# 既存ファイルをバックアップ（任意・ロールバック用）
$backupPath = "C:/deploy/backup/DevNext_$(Get-Date -Format 'yyyyMMdd_HHmmss')"
Copy-Item -Path "C:/inetpub/wwwroot/DevNext" -Destination $backupPath -Recurse
Write-Host "バックアップ完了: $backupPath"

# 新しいファイルをコピー
Copy-Item -Path "C:/deploy/DevNext/*" -Destination "C:/inetpub/wwwroot/DevNext" -Recurse -Force
Write-Host "ファイル配置完了"
```

**注意**: `Uploads/` フォルダ（ユーザーがアップロードしたファイル）は上書き対象から除外すること。

```powershell
# Uploads フォルダは除外してコピー（ユーザーデータ保護）
robocopy "C:/deploy/DevNext" "C:/inetpub/wwwroot/DevNext" /E /XD Uploads
```

---

### Step 4: 設定ファイルの確認

本番環境固有の設定を確認・更新する。

**`appsettings.json`** の確認項目：

```json
{
  "ConnectionStrings": {
    "SiteConnection": "Server=<本番サーバー>;Database=DevNextDB;..."
  },
  "Smtp": {
    "Host": "smtp.example.com",
    "Port": 25,
    "EnableSsl": true
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information"
    }
  }
}
```

> **注意**: `appsettings.json` には機密情報（接続文字列・パスワード）を含める場合があるため、本番サーバー上のファイルを直接編集するか、環境変数または `appsettings.Production.json` で上書きすること。

---

### Step 5: AppPool の起動

```powershell
Start-WebAppPool -Name 'DevNext'
Write-Host "AppPool 起動完了"

# 起動確認
Start-Sleep -Seconds 3
Get-WebAppPoolState -Name 'DevNext'
# 出力: Started
```

---

### Step 6: 動作確認

ブラウザまたは curl でアクセスを確認する。

```powershell
# HTTP レスポンス確認
$response = Invoke-WebRequest -Uri "http://localhost/DevNext" -UseBasicParsing
Write-Host "ステータスコード: $($response.StatusCode)"
# 期待値: 200 または 302（ログインページへリダイレクト）
```

## 4. DB 初期化・マイグレーション

### 新規環境セットアップ

初めてデプロイする環境（DB が存在しない場合）は `DbMigrationRunner` を実行する。

```bash
cd DbMigrationRunner
dotnet run
```

**実行内容**:
1. `DevNextDB` データベースの作成（`EnsureCreatedAsync`）
2. 既存 DB の場合は不足テーブル・カラムを差分適用（`ApplyMissingTablesAsync`）
3. 初期ロール（Admin / Member）の作成
4. 初期ユーザーの作成

**初期ユーザー**:

| UserName | Password | Role |
|----------|----------|------|
| admin1@sample.jp | `Admin1!` | Admin |
| member1@sample.jp | `Member1!` | Member |

---

### テーブル・カラム変更時

本プロジェクトは **マイグレーションファイルを使用しない**。テーブル構造の変更は以下の方法で反映する。

| 変更内容 | 手順 |
|---------|------|
| 新しいテーブルの追加 | `DbMigrationRunner/Program.cs` の `[テーブル追加]` セクションに CREATE TABLE ブロックを追記し、`DbMigrationRunner` を再実行 |
| 既存テーブルへのカラム追加 | `DbMigrationRunner/Program.cs` の `[カラム追加]` セクションに ALTER TABLE ブロックを追記し、`DbMigrationRunner` を再実行 |
| インデックスの変更 | SQL スクリプトを直接実行 |

> **注意**: `EnsureCreatedAsync` は新規 DB 作成時のみ動作する。既存 DB への差分適用は `ApplyMissingTablesAsync` が担う。
> テーブル・カラムを追加した場合は `[テーブル追加]` / `[カラム追加]` 両セクションへの追記が必要。

## 5. 動作確認

デプロイ完了後に以下を確認する。

### チェックリスト

| 確認項目 | 方法 |
|---------|------|
| トップページが表示される | ブラウザでアクセス |
| ログインできる | admin1@sample.jp / Admin1! でログイン |
| DB 接続が正常 | ホーム画面のダッシュボードにデータが表示される |
| ファイルアップロードが動作する | テストファイルをアップロード |
| メール送信が動作する | メール送信サンプルから送信テスト |
| 承認ワークフローが動作する | 申請作成・承認の一連フロー確認 |

## 6. ロールバック手順

問題が発生した場合、バックアップから復元する。

```powershell
# AppPool 停止
Stop-WebAppPool -Name 'DevNext'

# バックアップから復元
$backupPath = "C:/deploy/backup/DevNext_<バックアップ日時>"
Copy-Item -Path "$backupPath/*" -Destination "C:/inetpub/wwwroot/DevNext" -Recurse -Force
Write-Host "ロールバック完了"

# AppPool 起動
Start-WebAppPool -Name 'DevNext'
```

## 7. トラブルシューティング

### 502.5 / 500 エラー

| 原因 | 確認方法 | 対処 |
|------|---------|------|
| .NET ランタイム未インストール | `dotnet --list-runtimes` | .NET Hosting Bundle を再インストール |
| web.config の設定不正 | IIS のエラーログ | `aspNetCore` ハンドラーを確認 |
| 接続文字列エラー | Windows イベントログ | `appsettings.json` の DB 接続文字列を確認 |

### AppPool がすぐ停止する

| 原因 | 確認方法 | 対処 |
|------|---------|------|
| アプリ起動エラー | `C:/inetpub/wwwroot/DevNext/logs/stdout_*.log` | stdout ログを確認 |
| 依存ライブラリ不足 | イベントビューアー | Publish 時に `--self-contained` を使用 |

### stdout ログを有効化する方法

`web.config` の `stdoutLogEnabled` を `true` に変更する（デバッグ時のみ）：

```xml
<aspNetCore processPath=".\DevNext.exe"
            stdoutLogEnabled="true"
            stdoutLogFile=".\logs\stdout" />
```

> ログ出力後は必ず `false` に戻すこと（パフォーマンス影響あり）。
