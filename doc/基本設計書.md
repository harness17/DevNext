## 1. システム概要

DevNext は、ASP.NET Core 10 MVC をベースとした **エンタープライズ向け Web アプリケーションテンプレート**である。
旧 .NET Framework 版（DevNet）からの移行・リニューアルプロジェクトとして設計されており、業務案件で共通して必要となる機能を一通り実装した再利用可能な基盤を提供する。

### システム名

| 項目 | 内容 |
|------|------|
| システム名 | DevNext |
| システム種別 | Web アプリケーション（MVC） |
| 稼働環境 | Windows / Linux（.NET 10.0） |
| データベース | SQL Server |
| 想定利用者 | 認証済みユーザー（Admin / Member） |

## 2. 目的・背景

### 背景

実際の業務案件で得た知見をもとに、汎用的に再利用できる Web アプリケーション基盤として整理したもの。
「次の案件でもそのまま使える」ことを意識し、認証・CRUD・共通ライブラリなど、どの案件でも必要になる機能を一通り組み込んでいる。

### 目的

- 業務 Web アプリケーション開発の初期コスト削減
- 共通機能（認証・CRUD・ファイル管理・帳票出力）の標準化
- ASP.NET Core 10 の最新構成をベースにした保守性の高いコードベース提供
- CommonLibrary による横断的機能の再利用促進

### 設計方針

- **案件ごとの拡張を前提とした設計**：特定のビジネスロジックに依存しない構造
- **CommonLibrary の切り出し**：アプリ本体と共通機能を分離し、別プロジェクトへの流用を容易化
- **保守性の優先**：リポジトリ・サービス層・ViewModel を明確に分離し、影響範囲を局所化
- **テスト可能な設計**：DI・モックを最初から意識した構造化

## 5. アーキテクチャ概要

### レイヤードアーキテクチャ

本システムは以下の 5 層構造を採用する。

```
┌─────────────────────────────────────────────────────┐
│  Presentation Layer（プレゼンテーション層）             │
│  Controllers / Views / ViewModels                    │
├─────────────────────────────────────────────────────┤
│  Business Logic Layer（ビジネスロジック層）             │
│  Services                                            │
├─────────────────────────────────────────────────────┤
│  Data Access Layer（データアクセス層）                  │
│  Repositories                                        │
├─────────────────────────────────────────────────────┤
│  Domain Layer（ドメイン層）                            │
│  Entities                                            │
├─────────────────────────────────────────────────────┤
│  Cross-cutting Concerns（横断的関心事）                 │
│  CommonLibrary（Logger, Attributes, Extensions等）   │
└─────────────────────────────────────────────────────┘
```

### リクエスト処理フロー

```
クライアント
    │
    ▼
[HTTPS Redirect]
    │
    ▼
[Static Files]
    │
    ▼
[Routing]
    │
    ▼
[Session]
    │
    ▼
[Authentication / Authorization]
    │
    ▼
[Controller] ─── [AccessLogAttribute]
    │
    ▼
[Service]
    │
    ▼
[Repository]
    │
    ▼
[DBContext (Entity Framework Core)]
    │
    ▼
[SQL Server]
```

### 主なデザインパターン

| パターン | 適用箇所 |
|----------|---------|
| Repository パターン | データアクセス層（RepositoryBase + 各 Repository） |
| Service Layer パターン | ビジネスロジック層（各 Service） |
| Dependency Injection | Program.cs での DI コンテナ登録 |
| AutoMapper | Entity ↔ ViewModel の変換 |
| Soft Delete | DelFlag による論理削除 |
| Generic Programming | RepositoryBase\<TEntity, TCondModel\> |
| Filter / Middleware | アクセスログ（AccessLogAttribute） |

## 8. 非機能要件

### セキュリティ

| 項目 | 設定・方針 |
|------|-----------|
| 認証方式 | Cookie 認証（ASP.NET Core Identity） |
| セッション有効期限 | 1440 分（24 時間） |
| Cookie 設定 | HttpOnly、Secure 属性付き |
| パスワードハッシュ | ASP.NET Core Identity の PBKDF2 ハッシュ |
| アカウントロック | 5 回失敗 → 5 分ロック |
| HTTPS | HTTPS リダイレクト強制 |
| ファイルアップロード | サイズ・拡張子チェック、GUID ベースのファイル名でパストラバーサル防止 |
| CSRF 対策 | ASP.NET Core 標準の Antiforgery トークン |
| SQL インジェクション対策 | EF Core パラメータバインディング |

### 可用性・拡張性

| 項目 | 方針 |
|------|------|
| スケーラビリティ | ステートレス設計（セッションはサーバー側管理、必要に応じて分散セッションへ移行可能） |
| データ保護 | ASP.NET Core Data Protection（ファイルベースのキー永続化） |
| バッチ実行 | Mutex による多重起動防止 |

### 運用・保守

| 項目 | 方針 |
|------|------|
| ログ | アクセスログ（AccessLogAttribute）＋ ASP.NET Core 標準ログ |
| 論理削除 | 全エンティティで DelFlag による論理削除（物理削除なし） |
| 履歴管理 | SampleEntity / SampleEntityChild の更新履歴を History テーブルに保持 |
| テスト | xUnit / Moq によるユニットテスト |

## 9. 環境構成

### 前提条件

| 項目 | 要件 |
|------|------|
| .NET SDK | 10.0 以上 |
| データベース | SQL Server（localhost で起動済み） |
| SQL ユーザー | `admin` / `admin`（SQL Server 認証） |

### 接続設定

```json
{
  "ConnectionStrings": {
    "SiteConnection": "Server=localhost;Database=DevNextDB;Integrated Security=False;User ID=admin;Password=admin;TrustServerCertificate=True;"
  }
}
```

### 環境別設定

| 設定項目 | 開発環境 | 本番環境 |
|----------|---------|---------|
| ログレベル | Debug | Information |
| SMTP ホスト | localhost（smtp4dev） | smtp.example.com |
| SMTP ポート | 1025 | 25 |
| SSL | 無効 | 有効 |

### 起動手順

```bash
# 1. データベース作成・Seed 投入
cd DbMigrationRunner
dotnet run

# 2. Web アプリ起動
cd DevNext
dotnet run

# 3. テスト実行
cd Tests
dotnet test
```

### URL

| 環境 | URL |
|------|-----|
| 開発環境 | https://localhost:5001 |

### 初期ユーザー

| UserName | Password | Role |
|----------|----------|------|
