## 1. 画面共通仕様

### レイアウト

| 要素 | 内容 |
|------|------|
| フレームワーク | Bootstrap 5.3.8 |
| 共通レイアウト | `_Layout.cshtml`（ナビバー・フッター） |
| ナビバー | ロゴ、機能メニュー、通知ベルアイコン、ユーザーメニュー |
| フッター | コピーライト表示 |

### ナビバー構成

| メニュー | リンク先 | 表示条件 |
|---------|---------|---------|
| ホーム | /Home/Index | 認証済み |
| DB サンプル | /DatabaseSample/Index | 認証済み |
| ファイル管理 | /FileManagement/Index | 認証済み |
| メール送信 | /MailSample/Index | 認証済み |
| ウィザード | /WizardSample/Step1 | 認証済み |
| ダッシュボード | /Dashboard/Index | 認証済み |
| 承認申請 | /ApprovalRequest/Index | 認証済み |
| ユーザー管理 | /UserManagement/Index | Admin ロールのみ |
| 通知ベルアイコン | ドロップダウン表示 | 認証済み |
| ユーザーメニュー | アカウント設定・ログアウト | 認証済み |

### 通知ベルアイコン

- 未読通知がある場合、件数バッジを表示
- クリックでドロップダウン展開、最近の通知一覧を表示
- 通知クリックで関連ページへ遷移し、同時に既読処理
- 「すべて既読」ボタンで一括既読処理

### バリデーション

- クライアントサイド：jQuery Validation による即時バリデーション
- サーバーサイド：ASP.NET Core `ModelState` による検証
- エラーメッセージは対象フィールドの直下に赤文字で表示

### フラッシュメッセージ

- `TempData["SuccessMessage"]` / `TempData["ErrorMessage"]` で一時メッセージを制御
- 画面上部にアラートとして表示し、自動消去

## 2. 認証・アカウント管理画面

### ACC-001: ログイン画面

**URL**: `/Account/Login`
**権限**: 全員（未認証）

| 要素 | 内容 |
|------|------|
| メールアドレス入力 | 必須、メール形式 |
| パスワード入力 | 必須、マスク表示 |

## 3. 承認ワークフロー画面

承認ワークフローは本システムの中心機能であり、申請者（Member）と承認者（Admin）の役割が明確に分かれる。

### ステータス遷移

```
[下書き保存]
     ↓
 Draft（下書き）
     │
     │ [申請]
     ▼
 Pending（承認待ち）
     │
     ├─ [承認（Admin）] ──▶ Approved（承認済み）
     │
     └─ [却下（Admin）] ──▶ Rejected（却下）
```

| ステータス | 表示色 | 操作可能者 |
|-----------|--------|-----------|
| Draft | グレー | 申請者（編集・削除・申請） |
| Pending | 黄色 | Admin（承認・却下）、申請者（閲覧のみ） |
| Approved | 緑 | 誰も変更不可 |
| Rejected | 赤 | 誰も変更不可 |

---

### APR-001: 承認申請一覧画面

**URL**: `/ApprovalRequest/Index`
**権限**: 認証済み

**表示内容の権限差異**:

| ユーザー | 表示内容 |
|---------|---------|
| Admin | 全ユーザーの申請一覧 |
| Member | 自分が作成した申請のみ |

**画面要素**:

| 要素 | 内容 |
|------|------|
| 検索フォーム | タイトル・ステータス・期間で絞り込み |
| 申請一覧テーブル | ID・タイトル・申請者・ステータス・申請日・操作 |
| ページング | 10件/ページ |
| 新規作成ボタン | /ApprovalRequest/Create へ遷移 |
| CSV エクスポートボタン | 現在の検索条件で全件 CSV ダウンロード |
| Excel エクスポートボタン | 現在の検索条件で全件 Excel ダウンロード |

**操作列の表示ルール**:

| ステータス | 申請者の操作 | Admin の操作 |
|-----------|------------|------------|
| Draft | 編集・削除・申請 | 詳細 |
| Pending | 詳細 | 詳細・承認・却下 |
| Approved | 詳細 | 詳細 |
| Rejected | 詳細 | 詳細 |

---

### APR-002: 承認申請作成画面

**URL**: `/ApprovalRequest/Create`
**権限**: 認証済み

| 要素 | 内容 |
|------|------|
| タイトル | 必須、最大100文字 |
| 内容 | 必須、テキストエリア |
| 下書き保存ボタン | Draft ステータスで保存 |
| 申請するボタン | Pending ステータスで保存 → Admin 全員に通知 |

**申請時の通知**: 「新しい承認申請が提出されました（タイトル: 〇〇）」が Admin ロールを持つ全ユーザーに通知される。

---

### APR-003: 承認申請編集画面

**URL**: `/ApprovalRequest/Edit/{id}`
**権限**: 自分の Draft 申請のみ編集可能

| 要素 | 内容 |
|------|------|
| タイトル | 変更可能 |
| 内容 | 変更可能 |
| 下書き保存ボタン | Draft のまま保存 |
| 申請するボタン | Pending に変更して保存 |

**アクセス制御**:
- 他のユーザーの申請 → 403 返却
- Pending / Approved / Rejected の申請 → 編集不可、一覧にリダイレクト

---

### APR-004: 承認申請詳細・承認/却下画面

**URL**: `/ApprovalRequest/Detail/{id}`
**権限**: 認証済み

| 要素 | 内容 |
|------|------|
| 申請情報 | タイトル・内容・申請者・ステータス・申請日 |
| コメント履歴 | Admin のコメント一覧 |
| 承認ボタン（Admin のみ） | コメント入力後に承認処理 → 申請者に通知 |
| 却下ボタン（Admin のみ） | コメント入力後に却下処理 → 申請者に通知 |
| コメント入力欄 | 任意テキスト（承認・却下時のコメント） |

**承認/却下時の通知**: 「申請『〇〇』が承認（または却下）されました」が申請者に通知される。

---

### APR-005: 承認申請削除確認画面

**URL**: `/ApprovalRequest/Delete/{id}`
**権限**: 自分の Draft 申請のみ削除可能

| 要素 | 内容 |
|------|------|
| 申請情報確認 | 削除対象の申請内容を表示 |
| 削除確認ボタン | 論理削除実行 |
| キャンセルボタン | 一覧に戻る |

## 6. 多段階フォーム（ウィザード）画面

複数ステップで情報を入力し、確認・登録するフロー。セッションでステップ間データを保持する。

### 画面フロー

```
Step1（基本情報） → Step2（詳細情報） → 確認画面 → 完了画面
        ↑__________↑（「戻る」ボタン）
```

| 画面 | URL | 入力内容 |
|------|-----|---------|
| WIZ-001 Step1 | /WizardSample/Step1 | 氏名・メールアドレス・電話番号 |
| WIZ-002 Step2 | /WizardSample/Step2 | 件名・内容・カテゴリ・希望日 |
| WIZ-003 確認 | /WizardSample/Confirm | 入力内容の確認（変更不可） |
| WIZ-004 完了 | /WizardSample/Complete | 登録完了メッセージ・ホームへのリンク |

**セッション管理**:
- 各ステップの入力値をサーバーサイドセッションに保存
- 確認画面で全データを表示し、DB に保存
- 完了後はセッションデータをクリア
