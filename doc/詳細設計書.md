### サービス一覧

| クラス名 | 対応機能 | DI 登録スコープ |
|----------|---------|--------------|
| `DatabaseSampleService` | CRUD・Excel/PDF 出力 | Scoped |

### 設計方針

- サービスクラスはコンストラクターで `DBContext` を受け取り、内部でリポジトリをインスタンス化
- AutoMapper を使用して Entity ↔ ViewModel を変換
- `MapperConfiguration` は各サービス内でローカルに生成（グローバル Profile ではなく局所的に定義）
- AutoMapper 16.x では `NullLoggerFactory.Instance` を第 2 引数に渡す

```csharp
// AutoMapper 設定例
var config = new MapperConfiguration(cfg =>
{
    cfg.CreateMap<SampleEntity, SampleDetailViewModel>();
}, NullLoggerFactory.Instance);
var mapper = config.CreateMapper();
```

### リポジトリ一覧

| クラス名 | 対象エンティティ | 基底クラス |
|----------|---------------|----------|
| `SampleEntityRepository` | `SampleEntity` / `SampleEntityHistory` | `RepositoryBase<SampleEntity, SampleSearchModel>` |

| `FileEntityRepository` | `FileEntity` | `RepositoryBase<FileEntity, FileSearchModel>` |

### RepositoryBase（CommonLibrary）の提供メソッド

| メソッド | 説明 |
|---------|------|
| `SelectById(id)` | PK で 1 件取得 |
| `Select(condition)` | 条件検索（ページング・ソート対応） |
| `Insert(entity)` | 1 件登録（`SetForCreate()` を自動呼び出し） |
| `InsertSimple(entity)` | 監査情報なしで登録 |
| `BatchInsert(entities)` | 一括登録 |
| `Update(entity)` | 更新（`SetForUpdate()` を自動呼び出し） |
| `UpdateSimple(entity)` | 監査情報なしで更新 |
| `PhysicalDelete(id)` | 物理削除 |
| `LogicalDelete(id)` | 論理削除（`SetForLogicalDelete()` を呼び出し） |

### 履歴管理

- `SampleEntity` 更新・削除時は `SampleEntityHistory` テーブルに旧データをコピー
- `SampleEntityChild` も同様に `SampleEntityChildHistory` テーブルで履歴管理
- 履歴テーブルは `HistoryId`（bigint）を主キーとし、元テーブルと同じカラムを保持

## 5. エンティティ設計

### 継承階層

```
EntityBase（CommonLibrary）
    ├── DelFlag：bool
    ├── CreateDate：DateTime
    ├── UpdateDate：DateTime
    ├── CreateApplicationUserId：string?
    └── UpdateApplicationUserId：string?
        │
        └── SiteEntityBase（Site）
                └── Id：long（PK）
                    │
                    ├── SampleEntityBase（abstract）
                    │       ├── SampleEntity
                    │       └── SampleEntityHistory（+ HistoryId）
                    │
                    ├── SampleEntityChildBase（abstract）
                    │       ├── SampleEntityChild
                    │       └── SampleEntityChildHistory（+ HistoryId）
                    │
                    ├── FileEntity
                    └── WizardEntity
```

### EntityBase メソッド

| メソッド | 処理内容 |
|---------|---------|
| `SetForCreate()` | `CreateDate = UpdateDate = DateTime.Now`、ログイン中ユーザー ID を設定 |
| `SetForUpdate()` | `UpdateDate = DateTime.Now`、ログイン中ユーザー ID を更新 |
| `SetForLogicalDelete()` | `DelFlag = true`、`SetForUpdate()` を実行 |

ユーザー ID の取得には `IHttpContextAccessor` を静的プロパティで保持し、HTTP コンテキストから `ClaimTypes.NameIdentifier` を参照する。

### ApplicationUser（Identity 拡張）

ASP.NET Core Identity の `IdentityUser` を継承し、以下のフィールドを追加：

| フィールド | 型 | 説明 |
|-----------|---|------|
| `PreviousUserPasswords` | `IList<UserPreviousPassword>` | パスワード履歴 |
| `ResetPasswordTimeOut` | `DateTime?` | パスワードリセットのタイムアウト日時 |
| `PasswordAvailableEndDate` | `DateTime?` | パスワード有効期限 |
| `ApplicationRoleName` | `string?` | ロール名（表示用） |
| `UpdateApplicationUserId` | `string?` | 最終更新者 ID |

## 7. 認証・認可設計

### 認証フロー

```
1. ユーザーがログイン画面にアクセス
2. メールアドレス・パスワードを入力
3. SignInManager.PasswordSignInAsync() で認証
4. 成功 → Cookie 発行（有効期限：24 時間）
5. 失敗 → エラーメッセージ表示
6. 5 回失敗 → 5 分間アカウントロック
```

### ロール定義

| ロール名 | 表示名 | 概要 |
|---------|--------|------|
| `Admin` | 管理者 | 全機能にアクセス可能 |
| `Member` | 運営者 | 一般機能にアクセス可能（ユーザー管理を除く） |

### 認可方針

- 全コントローラー（Account を除く）は `[Authorize]` を付与
- `Account` コントローラーは `[AllowAnonymous]`
- ユーザー管理系の操作は `UserManagementService` 内でロールチェック

### Cookie 設定

```csharp
options.Cookie.HttpOnly = true;           // JavaScript からアクセス不可
options.Cookie.SecurePolicy = CookieSecurePolicy.Always; // HTTPS のみ
options.ExpireTimeSpan = TimeSpan.FromMinutes(1440); // 24 時間
options.SlidingExpiration = true;         // アクティブ時に延長
options.LoginPath = "/Account/Login";
options.AccessDeniedPath = "/Account/Login";
```

### パスワードポリシー

```csharp
options.Password.RequiredLength = 6;
options.Password.RequireUppercase = true;
options.Password.RequireLowercase = true;
options.Password.RequireDigit = true;
options.Password.RequireNonAlphanumeric = true;
```

### ロックアウト設定

```csharp
options.Lockout.MaxFailedAccessAttempts = 5;
options.Lockout.DefaultLockoutTimeSpan = TimeSpan.FromMinutes(5);
options.Lockout.AllowedForNewUsers = true;
```

## 9. ファイル管理設計

### ファイルアップロードフロー

```
1. クライアントがファイルをフォーム送信
2. FileAttribute でサイズ・拡張子を検証
3. GUID + 元の拡張子で保存ファイル名を生成
4. wwwroot/Uploads/[サブディレクトリ]/ に物理保存
5. FileEntity にメタ情報（元ファイル名、保存ファイル名、サイズ、ContentType）を DB 登録
```

### ファイル種別と保存先

| 種別 | 保存先 | 説明 |
|------|--------|------|
| サンプルエンティティ添付 | `~/Uploads/Sample/{EntityId}/` | エンティティへのファイル添付 |
| ファイル管理 | `~/Uploads/Files/` | ファイル管理機能のアップロード先 |

### セキュリティ対策

- **GUID ファイル名**：元のファイル名を使用しないことでパストラバーサルを防止
- **拡張子チェック**：`FileAttribute` で許可拡張子のみ受け付け
- **サイズ制限**：`FileAttribute` で最大サイズを制限
- **ContentType 記録**：ダウンロード時の適切なレスポンスヘッダー設定に使用

### ダウンロードフロー

```
1. クライアントがダウンロードリクエスト
2. FileEntity から SavedFileName を取得
3. 物理ファイルのパスを解決
4. FileResult でストリーミング送信（ContentType・OriginalFileName を設定）
```

## 10. 帳票出力設計

### Excel 出力（EPPlus）

| 種別 | 説明 |
|------|------|
| テンプレートベース出力 | `~/Templates/Excel/` 配下のテンプレートを読み込み、値を埋め込んで出力 |
| インポート | XLSX ファイルを読み込み、エンティティに変換して一括登録 |

**出力フロー（エクスポート）**

```
1. テンプレートファイルを読み込み
2. DB からデータ取得
3. EPPlus で値をセルに書き込み
4. MemoryStream に書き出し
5. FileContentResult でクライアントへ送信
```

**インポートフロー**

```
1. アップロードされた XLSX を EPPlus で読み込み
2. 各行をエンティティに変換
3. バリデーション（必須項目・データ型）
4. BatchInsert で一括 DB 登録
5. 結果（成功件数・エラー行）を画面に表示
```

### PDF 出力（QuestPDF）

| 種別 | 説明 |
|------|------|
| 一覧 PDF | 検索結果を表形式で出力 |
| 単体 PDF | 詳細情報 + 子エンティティ一覧 + 添付画像を出力 |

**画像埋め込み対応拡張子**：PNG、JPEG、JPG

**出力フロー**

```
1. DB からデータ取得
2. QuestPDF の Document クラスで帳票レイアウト定義
3. MemoryStream に PDF 生成
4. FileContentResult でクライアントへ送信（application/pdf）
```

```json
{
  "Smtp": {

    "Port": "25",
    "EnableSsl": "true",
```

### 送信フロー

- `smtp4dev`（ローカル擬似 SMTP サーバー）を使用
- ポート：1025
- `smtp4dev-start.ps1` で起動

## 12. バッチ処理設計

### BatchSample プロジェクト

コンソールアプリケーションとして実装。

### 処理フロー

```
1. BatchService.Run(IBatch) を呼び出し
2. Mutex を取得（多重起動チェック）
3. IBatch.Exec() を実行
4. 例外発生時は IBatch.ExceptionHandler(ex) を呼び出し
5. Mutex を解放
```

### IBatch 実装例

```csharp
public class SampleBatch : IBatch
{
    public async Task Exec()
    {
        // バッチ処理本体
    }

    public async Task ExceptionHandler(Exception ex)
    {
```

## 14. エラーハンドリング設計

### エラー種別

| エラー種別 | Enum 値 | 説明 |
|-----------|--------|------|
| システムエラー | `syserror` | 予期しないエラー |
| 不正な URL エラー | `urlerror` | 存在しないリソースへのアクセス |
| 不正な操作 | `usererror` | 権限外操作等 |
| セッションタイムアウト | `sessionerror` | セッション切れ |
| 使用できない機能 | `cannotuseerror` | 利用不可機能へのアクセス |

### エラー処理フロー

```
1. 例外発生 or 不正リクエスト
2. ミドルウェアがキャッチ
3. RootErrorController.Error() へリダイレクト
4. ErrorType に応じたメッセージを表示
```

### 開発環境

- 開発環境では `app.UseDeveloperExceptionPage()` でスタックトレースを表示
