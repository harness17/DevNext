## 1. テスト方針

### 基本方針

- **ユニットテストを主体**とし、外部依存（DB・SMTP等）はモックで代替する
- **CommonLibrary の共通機能**はテスト可能な純粋関数として設計し、単体テストで網羅する
- **テスト可能な設計（Testable Design）** を前提とし、DI によって依存を差し替え可能にする
- 境界値・異常系を含む複数パターンを `[Theory]` / `[InlineData]` でまとめて検証する

### テスト対象の優先度

| 優先度 | 対象 | 理由 |
|--------|------|------|
| 高 | CommonLibrary の拡張メソッド・ユーティリティ | 全機能で横断的に使用されるため、バグの影響範囲が広い |
| 高 | サービス層のビジネスロジック（状態遷移・認可） | 承認ワークフローなど複雑なロジックを含む |
| 中 | リポジトリ層 | EF Core に大きく依存するため、統合テストが必要 |
| 低 | コントローラー | UI ロジックはミニマムに保ち、サービス層に委譲 |

## 2. テスト範囲

### テスト対象

| プロジェクト | テスト対象 | テスト種別 |
|-------------|-----------|-----------|
| CommonLibrary | StringExtensions | ユニットテスト |
| CommonLibrary | Util（共通関数） | ユニットテスト |
| CommonLibrary | EnumUtility | ユニットテスト（今後追加予定） |
| DevNext | サービス層 | ユニットテスト（今後追加予定） |

### テスト対象外

| 対象 | 除外理由 |
|------|---------|
| Razor ビュー | UI テストは agent-browser による動作確認で対応 |
| DB アクセス（リポジトリ） | EF Core の統合テストは別途検討 |
| 外部メールサーバー | 開発環境では smtp4dev で代替 |

### 4.1 StringExtensions テスト

**ファイル**: `Tests/StringExtensionsTests.cs`
**テストクラス**: `StringExtensionsTests`
**対象クラス**: `Dev.CommonLibrary.Extensions.StringExtensions`

#### HtmlEncode / HtmlDecode

| テストメソッド | 入力 | 期待値 | 観点 |
|--------------|------|--------|------|
| `HtmlEncode_WithSpecialChars_ReturnsEncoded` | `<>&"` などの特殊文字 | HTMLエンティティに変換 | 正常系：XSS対策の変換確認 |
| `HtmlEncode_Null_ReturnsNull` | `null` | `null` | 異常系：null安全性 |
| `HtmlDecode_WithEntities_ReturnsDecoded` | `&lt;&gt;&amp;` など | 元の文字に復元 | 正常系：デコード確認 |
| `HtmlDecode_Null_ReturnsNull` | `null` | `null` | 異常系：null安全性 |

#### GetBrText

| テストメソッド | 入力 | 期待値 | 観点 |
|--------------|------|--------|------|
| `GetBrText_ReplacesNewlinesWithBrTag` | 改行を含む文字列（複数パターン） | `\n` → `<br>` に置換 | 正常系：HTML表示用変換 |
| `GetBrText_Null_ReturnsEmpty` | `null` | 空文字 | 異常系：null安全性 |

#### ContainsAll / ContainsAny

| テストメソッド | 入力 | 期待値 | 観点 |
|--------------|------|--------|------|
| `ContainsAll_AllValuesPresent_ReturnsTrue` | 全て含む文字列 | `true` | 正常系：全一致 |
| `ContainsAll_SomeValuesMissing_ReturnsFalse` | 一部欠ける文字列 | `false` | 正常系：部分不一致 |
| `ContainsAny_OneValuePresent_ReturnsTrue` | 1つ以上含む文字列 | `true` | 正常系：いずれか一致 |
| `ContainsAny_NonePresent_ReturnsFalse` | 一つも含まない文字列 | `false` | 正常系：全不一致 |

#### Left / Right / Mid

| テストメソッド | 入力 | 期待値 | 観点 |
|--------------|------|--------|------|
| `Left_ReturnsExpected` | 文字列と文字数（複数パターン） | 左からN文字 | 正常系・境界値 |
| `Right_ReturnsExpected` | 文字列と文字数（複数パターン） | 右からN文字 | 正常系・境界値 |

## 5. テスト観点

### 5.1 セキュリティ観点

| 観点 | 対象 | テストで確認している内容 |
|------|------|------------------------|
| XSS 対策 | `HtmlEncode` | 特殊文字がエンティティに変換されることを確認 |
| パストラバーサル | `IsSafePath` | `../` などの危険なパスを拒否することを確認 |

### 5.2 境界値観点

| 観点 | 対象 | テストで確認している内容 |
|------|------|------------------------|
| 文字列切り出し | `Left` / `Right` / `Mid` | 0文字・最大文字数・超過文字数 |
| ページング | `CreateSummary` | 先頭ページ・最終ページ・端数処理 |

### 5.3 null 安全性観点

| 対象 | 確認内容 |
|------|---------|
| `HtmlEncode` / `HtmlDecode` | null 入力でも例外が発生しない |
| `GetBrText` | null 入力で空文字を返す |
| `IsSafePath` | null / 空文字で false を返す |

## 6. テスト実行方法

### 基本実行

```bash
cd Tests
dotnet test
```

### 詳細出力

```bash
cd Tests
dotnet test --logger "console;verbosity=detailed"
```

### 特定クラスのみ実行

```bash
cd Tests
dotnet test --filter "FullyQualifiedName~StringExtensionsTests"
```

### カバレッジ計測（将来対応）

```bash
cd Tests
dotnet test --collect:"XPlat Code Coverage"
```

## 7. 今後のテスト拡張方針

### 優先度 高

| 対象 | 追加すべきテスト | 理由 |
|------|----------------|------|
| `ApprovalWorkflowService` | 状態遷移テスト（Draft→Pending→Approved/Rejected） | 複雑なビジネスロジックを含む |
| `ApprovalWorkflowService` | 権限チェック（Admin 以外が承認できないこと） | セキュリティ要件 |
| `NotificationService` | 通知生成ロジック | Admin 全員通知・申請者への通知 |

### 優先度 中

| 対象 | 追加すべきテスト | 理由 |
|------|----------------|------|
| `EnumUtility` | Enum 表示名取得・ドロップダウン生成 | 全画面で利用 |
| `DatabaseSampleService` | CRUD の正常系・異常系 | 基本機能の品質保証 |
| `ExportService` | CSV / Excel 出力内容の検証 | 帳票出力の正確性 |

### 優先度 低

| 対象 | 追加すべきテスト | 理由 |
|------|----------------|------|
| コントローラー | 認可チェック（未認証リダイレクト） | agent-browser 動作確認と重複するが念のため |
| `RepositoryBase` | インメモリDBを使った統合テスト | EF Core の動作確認 |
