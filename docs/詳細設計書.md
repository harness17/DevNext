### NotificationService 主要メソッド

| メソッド | 説明 |
|---------|------|
| `CreateAsync` | 1人のユーザーに通知を作成 |
| `CreateForMultipleAsync` | 複数ユーザーに同じ通知を一括作成（Admin への申請通知） |
| `GetUnreadCountAsync` | 未読件数を返す（ベルバッジ用） |
| `GetRecentAsync` | 最新通知を返す（最大10件） |
| `MarkAsReadAsync` | 指定 ID を既読に（本人の通知のみ） |
| `MarkAllAsReadAsync` | すべての未読を既読に（`ExecuteUpdateAsync` で一括更新） |

> **ポイント：** `CreateForMultipleAsync` は `InsertRangeAsync` で一括 INSERT する（N+1 回避）。`MarkAllAsReadAsync` は `ExecuteUpdateAsync` で全件一括 UPDATE する（N+1 回避）。

### ExportService 主要メソッド

| メソッド | 説明 |
|---------|------|
| `ExportCsvAsync` | 条件一致全件を CSV（BOM 付き UTF-8）に変換して返す |
| `ExportExcelAsync` | 条件一致全件を xlsx に変換して返す（ClosedXML 使用） |

### 設計方針

- サービスクラスは原則リポジトリを DI 注入で受け取る（`NotificationService` など）。一部サービス（`ScheduleService` 等）はリポジトリを内部でインスタンス化するパターンも混在
- AutoMapper を使用して Entity ↔ ViewModel を変換
- `MapperConfiguration` は各サービス内でローカルに生成（グローバル Profile ではなく局所的に定義）
- AutoMapper 16.x では `NullLoggerFactory.Instance` を第 2 引数に渡す

```csharp
// AutoMapper 設定例
var config = new MapperConfiguration(cfg =>
{
    cfg.CreateMap<SampleEntity, SampleDetailViewModel>();
}, NullLoggerFactory.Instance);
var mapper = config.CreateMapper();
```

- 履歴テーブルは `HistoryId`（bigint）を主キーとし、元テーブルと同じカラムを保持

## 7. 認証・認可設計

### 認証フロー

```
1. ユーザーがログイン画面にアクセス
2. メールアドレス・パスワードを入力
3. SignInManager.PasswordSignInAsync() で認証
4. 成功 → Cookie 発行（有効期限：24 時間）
5. 失敗 → エラーメッセージ表示
6. 5 回失敗 → 5 分間アカウントロック
```

### ロール定義

| ロール名 | 表示名 | 概要 |
|---------|--------|------|
| `Admin` | 管理者 | 全機能にアクセス可能 |
| `Member` | 運営者 | 一般機能にアクセス可能（ユーザー管理を除く） |

### 認可方針

- 全コントローラー（Account を除く）は `[Authorize]` を付与
- `Account` コントローラーは `[AllowAnonymous]`
- ユーザー管理系の操作は `UserManagementService` 内でロールチェック

### Cookie 設定

```csharp
options.Cookie.HttpOnly = true;           // JavaScript からアクセス不可
options.Cookie.SecurePolicy = CookieSecurePolicy.Always; // HTTPS のみ
options.ExpireTimeSpan = TimeSpan.FromMinutes(1440); // 24 時間
options.SlidingExpiration = true;         // アクティブ時に延長
options.LoginPath = "/Account/Login";
options.AccessDeniedPath = "/Account/Login";
```

### パスワードポリシー

```csharp
options.Password.RequiredLength = 6;
options.Password.RequireUppercase = true;
options.Password.RequireLowercase = true;
options.Password.RequireDigit = true;
options.Password.RequireNonAlphanumeric = true;
```

### ロックアウト設定

## 9. ファイル管理設計

### ファイルアップロードフロー

```
1. クライアントがファイルをフォーム送信
2. FileAttribute でサイズ・拡張子を検証
3. GUID + 元の拡張子で保存ファイル名を生成
4. wwwroot/Uploads/[サブディレクトリ]/ に物理保存
5. FileEntity にメタ情報（元ファイル名、保存ファイル名、サイズ、ContentType）を DB 登録
```

### ファイル種別と保存先

| 種別 | 保存先 | 説明 |
|------|--------|------|
| サンプルエンティティ添付 | `~/Uploads/Sample/{EntityId}/` | エンティティへのファイル添付 |
| ファイル管理 | `~/Uploads/Files/` | ファイル管理機能のアップロード先 |

### セキュリティ対策

- **GUID ファイル名**：元のファイル名を使用しないことでパストラバーサルを防止
- **拡張子チェック**：`FileAttribute` で許可拡張子のみ受け付け
- **サイズ制限**：`FileAttribute` で最大サイズを制限
- **ContentType 記録**：ダウンロード時の適切なレスポンスヘッダー設定に使用

### ダウンロードフロー

```
1. クライアントがダウンロードリクエスト
2. FileEntity から SavedFileName を取得
3. 物理ファイルのパスを解決
4. FileResult でストリーミング送信（ContentType・OriginalFileName を設定）
```

## 10. 帳票出力設計

### Excel 出力（DatabaseSample：EPPlus → ClosedXML 移行済み）

> **注意：** 旧 EPPlus から MIT ライセンスの **ClosedXML** に移行済み。商用利用でも無償使用可能。

| 種別 | 説明 |
|------|------|
| テンプレートベース出力 | `~/Templates/Excel/` 配下のテンプレートを読み込み、値を埋め込んで出力 |
| インポート | XLSX ファイルを読み込み、エンティティに変換して一括登録 |

**出力フロー（エクスポート）**

```
1. テンプレートファイルを読み込み
2. DB からデータ取得
3. ClosedXML で値をセルに書き込み
4. MemoryStream に書き出し
5. FileContentResult でクライアントへ送信
```

**インポートフロー**

```
1. アップロードされた XLSX を ClosedXML で読み込み
2. 各行をエンティティに変換
3. バリデーション（必須項目・データ型）
4. BatchInsert で一括 DB 登録
5. 結果（成功件数・エラー行）を画面に表示
```

### CSV・Excel エクスポート（承認申請：ExportService）

承認申請の一覧エクスポートは `ExportService` が担当。ページングなしで検索条件一致の全件を出力する。

| 種別 | ライブラリ | 特徴 |
|------|----------|------|
| CSV | System.Text（標準） | BOM 付き UTF-8、Excel で開いても文字化けしない |
| Excel | ClosedXML | ヘッダー太字・背景色・列幅自動調整（AdjustToContents） |

**エクスポートフロー**

```
1. 検索条件モデルを組み立て（ページングなし）
2. DB から全件取得
3. 申請者 ID を一括収集し UserManager で名前解決（N+1 回避）
4. CSV / xlsx に変換して MemoryStream に書き出し
5. FileContentResult でクライアントへ送信
```

### PDF 出力（QuestPDF）

| 種別 | 説明 |
|------|------|
| 一覧 PDF | 検索結果を表形式で出力 |
| 単体 PDF | 詳細情報 + 子エンティティ一覧 + 添付画像を出力 |

**画像埋め込み対応拡張子**：PNG、JPEG、JPG

**出力フロー**

```
1. DB からデータ取得
2. QuestPDF の Document クラスで帳票レイアウト定義
3. MemoryStream に PDF 生成
4. FileContentResult でクライアントへ送信（application/pdf）
```

## 11. メール送信設計

### Email クラス

`Site.Common.Email` クラスが SMTP 送信を担当。

### 設定（appsettings.json）

```json
{
  "Smtp": {
    "Host": "smtp.example.com",
    "Port": "25",
    "EnableSsl": "true",
    "UserName": "",
    "Password": "",
    "Timeout": "30000"
  },
  "Mail": {
    "From": "noreply@example.com",
    "FromName": "DevNext",
    "AdminAddress": ""
  }
}
```

### 送信フロー

```
1. ViewModel からメール内容（宛先・件名・本文）取得
2. テンプレートファイル読み込み
3. プレースホルダー（{KEY} 形式）を値に置換
4. SmtpClient で送信
```

| クラス | 役割 |
